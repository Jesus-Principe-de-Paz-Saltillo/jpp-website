# Sitio Web – Mision Bautista Jesús Príncipe de Paz

Este repositorio contiene el código fuente del sitio web oficial de la iglesia **Jesús Príncipe de Paz**.
El objetivo del proyecto es ofrecer un sitio web sencillo, claro y accesible para comunicar información
relevante a la comunidad (eventos, horarios, contacto, etc.).

> Este proyecto inicia como un sitio web estático y podrá evolucionar conforme surjan nuevas necesidades.

---

## 1) ¿Para qué es este repositorio?

### 1.1) Resumen rápido

Este repositorio aloja el sitio web público de la iglesia **Jesús Príncipe de Paz**.
Actualmente está construido como un **sitio web estático** y enfocado en simplicidad,
claridad y facilidad de mantenimiento.

El sitio está pensado para:
- Informar a la comunidad sobre la iglesia.
- Publicar horarios, eventos y avisos.
- Servir como base para futuras mejoras visuales o técnicas.

### 1.2) Declaración
_Este proyecto no tiene fines comerciales.
No existe relación financiera directa o indirecta con entidades comerciales derivada de este repositorio.
El desarrollo se realiza de forma voluntaria para beneficio de la comunidad._

---

## 2) ¿Cómo se configura el proyecto?

### 2.1) Resumen de configuración
Este proyecto es un **sitio web estático**.

Tecnologías utilizadas actualmente:
- **HTML5**

No se requiere backend, base de datos ni instalación local de herramientas.

---

### 2.2) Forma recomendada de trabajo

#### 2.2.1) Edición del proyecto
La forma recomendada de trabajar **por el momento** es utilizando el **editor web de GitHub**:

1. Abre el repositorio en GitHub.
2. Presiona la tecla `.` (punto) para abrir el editor web.
3. Realiza los cambios directamente sobre los archivos HTML.
4. Crea una rama y un Pull Request para integrar los cambios.

Este enfoque:
- Evita configuraciones locales innecesarias.
- Facilita la colaboración.
- Es suficiente para el alcance actual del proyecto.

> El uso de editores locales (VS Code, IntelliJ, etc.) se deja como opción para etapas futuras,
cuando el proyecto crezca en complejidad.

---

#### 2.2.2) Protección de la rama `main`
La rama `main` representa la versión estable del sitio.

Reglas establecidas:
- `main` estará protegida.
- No se permite hacer push directo a `main`.
- Todos los cambios deben entrar mediante **Pull Requests**.

Esto garantiza:
- Historial claro de cambios.
- Revisión antes de publicar.
- Estabilidad del sitio público.

---

## 3) ¿Cómo se despliega el sitio?

El sitio se despliega mediante **GitHub Pages**.

### Proceso de despliegue
- GitHub Pages ejecuta automáticamente su proceso de build al hacer **merge a `main`**.
- No se requiere ejecutar comandos manuales.
- El sitio se actualiza automáticamente después de aprobar un Pull Request.

Requisitos:
- El archivo `index.html` debe existir en la raíz del repositorio (o en la ruta configurada).
- GitHub Pages debe estar habilitado en la configuración del repositorio.

---

## 4) ¿Cuáles son las guías de contribución?

### 4.1) Estilo de contenido y código
- HTML limpio y semántico.
- Evitar complejidad innecesaria.
- Priorizar claridad, accesibilidad y legibilidad.

### 4.2) Revisión de cambios
- Revisar visualmente los cambios antes de crear el Pull Request.
- Verificar que el sitio se vea correctamente en desktop y móvil.

### 4.3) Otras consideraciones
- Mantener el contenido alineado con los valores y propósito de la iglesia.
- Evitar dependencias o herramientas sin una necesidad clara.
- Cualquier sugerencia o mejora es bienvenida.
